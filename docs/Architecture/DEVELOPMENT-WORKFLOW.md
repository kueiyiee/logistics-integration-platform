# Development Workflow

## Day-to-day flow

1. Pull the latest `develop` branch.
2. Create a feature branch from `develop`.
3. Implement the smallest coherent slice of work.
4. Add tests alongside code.
5. Open a pull request with screenshots or API examples when relevant.
6. Review, fix, and merge after checks pass.

## Quality gates

- Static analysis where applicable.
- Feature and unit tests for changed behavior.
- API contract validation for integration endpoints.
- UI verification for portal pages and layout changes.
